```markdown
### 📝 Generación de Reportes Integrales

#### 🎯 Instrucciones de Procesamiento

1.  **✅ Selección de Contenido**
    Permitir al usuario seleccionar posts para el reporte mediante:
    - 🆔 **IDs** específicos.
    - 🔍 **Criterios de búsqueda** o filtros activos.
    - 📂 **Categorías** o 🏷️ **Tags**.
    - ⭐ **Grupos de Favoritos** (Soporta jerarquías/sub-grupos).

2.  **📄 Generación de Resumen**
    Estructurar un resumen detallado que incluya:
    - 📌 **Encabezado:** Títulos y URLs directas.
    - 🏷️ **Taxonomía:** Tags y categorías aplicadas.
    - ✨ **Contexto:** Resaltado de términos clave (integrado con `etiquetar`).
    - 📅 **Temporalidad:** Fecha y hora exacta de publicación.
    - 📊 **Métricas:** Puntuación (Score) y volumen de votos.
    - 💬 **Discusión:** Comentarios principales o más relevantes.

3.  **💾 Formatos de Exportación**
    Soportar la generación de archivos en múltiples formatos:
    - 🌐 **HTML:** Para visualización rich-text en navegador.
    - 📝 **Markdown:** Para documentación técnica y lecturabilidad.
    - 📊 **CSV:** Para análisis de datos estructurados.
    - 📄 **TXT:** Para portabilidad simple.
    - 💻 **JSON:** Para integración programática e interoperabilidad.

4.  **⚙️ Multi-idioma (i18n)**
    Los reportes pueden beneficiarse del esquema bilingüe del proyecto (🇪🇸 Español / 🇺🇸 Inglés), respetando la configuración del reporte si el usuario lo requiere.

5.  **⚙️ Personalización Modular**
    Habilitar secciones configurables por el usuario:
    - 📑 **Resumen General:** Visión de alto nivel.
    - 📎 **Posts Seleccionados:** Detalle granular.
    - 📈 **Estadísticas:** Análisis numérico del conjunto.
    - 🗨️ **Comentarios:** Extracto de la discusión comunitaria.
```
